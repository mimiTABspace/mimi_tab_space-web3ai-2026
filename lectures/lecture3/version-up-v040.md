# バージョンアップ内容 - Self Care Quest v0.4.0

**バージョン**: 0.3.0 → 0.4.0  
**更新日**: 2026-06-13

---

## 主要新機能

### カレンダー＋体調グラフページ（`/calendar`）

ナビバーの「ルート」を「カレンダー」に刷新し、新ページを新設。

**タブ1「カレンダー」**
- 月表示カレンダーで服薬記録・通院予定をまとめて確認
- 服薬: 飲んだ日に✅、飲めなかった過去日に❌アイコン
- 通院予定: 🏥アイコン表示
- 日付タップで詳細ポップアップ（病院名・薬の名前・時刻）
- 前月/翌月ナビゲーション

**タブ2「体調グラフ」**
- 気分・疲労度の推移を折れ線グラフで表示
- 1週間 / 1ヶ月の切り替え
- 同日複数記録は平均値で表示
- ライブラリ: `recharts`
- 既存の `health_logs` テーブルをそのまま利用

---

### Care Route をホームの Main Quest に統合

- ナビバーから「ルート」タブを削除
- ホームの Main Quest セクションに QuestCard として「Care Route」を常時表示
  - 治療中のルートがある場合: 「治療中: N件」
  - ない場合: 「治療中のルートなし」

---

### 通院予定の編集・削除

予定タブの各通院カードに操作ボタンを追加。

| ボタン | 動作 |
|--------|------|
| 受診完了 | 受診結果・次回予定を入力して完了記録 |
| 📅 日時・目的変更 | 受診日時と目的をその場で変更 |
| 🗑 削除 | 確認ダイアログ付きで予定を削除 |

## 機能改善

### 家族共有メンバーへのニックネーム設定

家族共有中のメンバーにニックネームを設定できるようになった。

**背景**
- これまで体調・服薬・通院ページで共有メンバーがメールアドレスで表示されており、誰のデータかわかりにくかった

**変更内容**
- 設定ページの「家族共有」セクションで各メンバーに名前をつけられるようになった
  - 例: 「お母さん」「パパ」など自由に設定
  - 設定済みの場合は「変更」ボタンで修正可能
  - 未設定の場合は従来通りメールアドレスを表示
- ニックネームは体調・服薬・通院ページのメンバー表示にも反映される

## 技術メモ

- カレンダーの日付判定は `YYYY-MM-DD` 文字列の前方一致でフィルタ
- 体調グラフは日付ごとに `mood` / `fatigue` を平均化して描画
- 日時変更モーダルは `datetime-local` 入力値をローカルタイムオフセット補正してから ISO 文字列に変換
- `recharts` を依存関係に追加
- TypeScript の `ignoreBuildErrors: true` 設定によりimport漏れがビルド時に検出されなかった（要注意）
- Supabase の `family_shares` テーブルに `nickname text` カラムを追加
- 設定ページでインライン編集UI（保存・キャンセル）を実装
- 体調・通院ページは `_ownerEmail` に `nickname || member_email` を格納
- 服薬ページはセクション見出しで `share.nickname || share.member_email` を表示
